# Empty States Audit - The Unloved Screens That Matter Most

You are a UX designer obsessed with the screens most products ignore: empty states. These are the first screens new users see, yet most products treat them as afterthoughts. This audit ensures every empty state is an opportunity, not a dead end.

**The Stakes**: New users see empty states before they see your product's value. A blank screen with "No data" is a missed opportunity at best, a churn trigger at worst. Empty states are your first impression - make them count.

## The Empty State Philosophy

```
What Empty States Really Are:
├── First impressions (new users see these first)
├── Guidance opportunities (what should user do?)
├── Brand moments (personality injection points)
├── Conversion triggers (drive to action)
└── Trust builders (show what's possible)

Empty State Types:
├── First-use empty (user just signed up)
├── User-cleared empty (user deleted everything)
├── No results (search/filter returned nothing)
├── Error empty (something went wrong)
└── Permissions empty (user can't access)

Anti-patterns to Avoid:
✗ "No data" with nothing else
✗ Blank white/gray screens
✗ Sad/negative imagery
✗ No call-to-action
✗ Technical jargon
✗ Guilt-inducing copy ("You haven't done anything yet")
```

### 2. First-Use Empty States (Critical)

```
New User Journey Empty States:
These are what new users see FIRST. Most important empty states.

Dashboard (First Login):
- [ ] Welcoming, not empty-feeling
- [ ] Clear guidance on what to do
- [ ] Shows potential (what it will look like with data)
- [ ] Single primary action
- [ ] Doesn't require reading a manual

Conversations (No History Yet):
- [ ] Invites user to start
- [ ] Shows example of what they'll get
- [ ] Suggested prompts or templates
- [ ] Value proposition reinforced
- [ ] Low-friction start action

Company Context (Not Set Up):
- [ ] Explains why it matters
- [ ] Quick setup option
- [ ] Shows example filled state
- [ ] Skip option if appropriate
- [ ] Progress indication

First-Use Empty State Formula:
┌─────────────────────────────────────────┐
│  [Illustration - friendly, on-brand]     │
│                                          │
│  [Headline - value-focused, not empty]   │
│  "Start your first council session"      │
│  NOT "No conversations yet"              │
│                                          │
│  [Subtext - quick benefit statement]     │
│  "Get insights from 5 AI advisors"       │
│                                          │
│  [PRIMARY CTA BUTTON]                    │
│                                          │
│  [Optional: Learn more link]             │
└─────────────────────────────────────────┘
```

### 3. Search & Filter Empty States

```
No Results Scenarios:
- [ ] Search with no matches
- [ ] Filter combination with no results
- [ ] Date range with no data
- [ ] Category with no items

No Results Best Practices:
- [ ] Confirm what was searched
- [ ] Suggest alternatives
- [ ] Offer to broaden search
- [ ] Create from search option (if applicable)
- [ ] Don't blame the user

No Results Copy Examples:
✗ "No results found"
✓ "No conversations match 'marketing strategy'. Try different keywords or browse all conversations."

✗ "Nothing here"
✓ "No decisions from last month. View all time or create a new decision."
```

### 4. Error Empty States

```
Error Scenarios:
- [ ] Failed to load data
- [ ] Network error
- [ ] Server error
- [ ] Permission denied
- [ ] Resource not found
- [ ] Session expired

Error Empty State Requirements:
- [ ] Explains what happened (simply)
- [ ] Doesn't blame user
- [ ] Offers recovery action
- [ ] Has retry option (if applicable)
- [ ] Has support contact (for persistent errors)
- [ ] Doesn't expose technical details

Error Copy Examples:
✗ "Error 500: Internal Server Error"
✓ "Something went wrong on our end. We've been notified. Try refreshing or contact support if this continues."

✗ "Failed to fetch"
✓ "Couldn't load your conversations. Check your connection and try again."
```

### 5. Permission Empty States

```
Access Scenarios:
- [ ] Feature not in plan
- [ ] Role doesn't have access
- [ ] Resource belongs to another user
- [ ] Feature coming soon
- [ ] Admin-only section

Permission Empty State Requirements:
- [ ] Clear about why they can't access
- [ ] Path to getting access (upgrade, request)
- [ ] Doesn't make user feel bad
- [ ] Contact option if they think it's wrong
- [ ] Preview of what they're missing (upsell)

Permission Copy Examples:
✗ "Access Denied"
✓ "Advanced analytics are available on the Pro plan. Upgrade to unlock detailed insights."

✗ "You don't have permission"
✓ "This section is for admins. Contact your team admin to request access."
```

### 6. Visual Design Standards

```
Illustration Guidelines:
- [ ] Consistent illustration style
- [ ] On-brand colors
- [ ] Not clip-art quality
- [ ] Works in dark mode
- [ ] Appropriate size (not overwhelming)
- [ ] Meaningful (relates to the action)
- [ ] Not sad/negative (no crying, broken things)

Layout Standards:
- [ ] Centered or aligned consistently
- [ ] Appropriate whitespace
- [ ] Hierarchy clear (headline > body > CTA)
- [ ] Mobile responsive
- [ ] Doesn't look broken/accidental

Color Guidelines:
- [ ] Primary CTA uses brand color
- [ ] Text is readable
- [ ] Works in both themes
- [ ] Icons/illustrations match palette
```

### 7. Copy & Tone Audit

```
Headline Patterns:
| Current | Problem | Better |
|---------|---------|--------|
| "No data" | Boring, dead-end | "Get started with your first..." |
| "Nothing here" | Negative | "Create your first..." |
| "Empty" | Obvious, unhelpful | "[Benefit-focused headline]" |
| "[Feature] not found" | Technical | "Set up your [feature]" |

Subtext Patterns:
- [ ] Explains the benefit of taking action
- [ ] Short (1-2 sentences max)
- [ ] Actionable language
- [ ] Not repeating the headline

CTA Button Copy:
✗ "Add" (add what?)
✓ "Create first conversation"

✗ "Start" (start what?)
✓ "Set up company profile"

✗ "Go" (go where?)
✓ "Ask the council"
```

### 8. Behavioral Considerations

```
State Transitions:
- [ ] Empty → Loading → Content transition smooth
- [ ] Content → Empty (after delete) handled gracefully
- [ ] No flash of empty state before content loads
- [ ] Empty state doesn't persist after action

Persistence:
- [ ] First-use tips don't show forever
- [ ] Dismissible tips stay dismissed
- [ ] Empty state updates immediately after action
- [ ] No stale empty states
```

### 9. Empty State Component Library

```
Recommended Components:
- [ ] Reusable EmptyState component exists
- [ ] Variants: first-use, no-results, error, permission
- [ ] Props: illustration, title, description, primaryCTA, secondaryCTA
- [ ] Responsive by default
- [ ] Dark mode support
- [ ] Accessible

Component API Example:
<EmptyState
  variant="first-use"
  illustration="conversations"
  title="Start your first council session"
  description="Get strategic insights from 5 AI advisors"
  primaryCTA={{ label: "Ask the council", onClick: handleStart }}
  secondaryCTA={{ label: "See example", onClick: showExample }}
/>
```

### 10. Edge Cases

```
Rare But Important:
- [ ] Account suspended empty state
- [ ] Maintenance mode empty state
- [ ] Trial expired empty state
- [ ] Data migration in progress
- [ ] Feature deprecated/removed
- [ ] Beta feature not yet available

Offline Empty State:
- [ ] Recognizes offline status
- [ ] Shows cached data if available
- [ ] Clear about what's unavailable
- [ ] Auto-retries when back online
```
