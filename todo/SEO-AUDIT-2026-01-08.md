### Technical Details

**robots.txt Configuration:**
```
User-agent: *
Allow: /
Disallow: /chat/
Disallow: /settings/
Disallow: /company/
Disallow: /leaderboard
Allow: /$
Sitemap: https://axcouncil.vercel.app/sitemap.xml
```

**Why This Configuration:**
- Public landing page (`/$`) is allowed for indexing
- Authenticated app routes are blocked (require login)
- Sitemap reference helps crawlers discover the structure

**sitemap.xml Configuration:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://axcouncil.vercel.app/</loc>
    <lastmod>2026-01-07</lastmod>
    <changefreq>weekly</changefreq>
    <priority>1.0</priority>
  </url>
</urlset>
```

**Why Only Landing Page:**
- All other pages require authentication
- Google cannot index authenticated content
- This is standard practice for SaaS apps

**JSON-LD Structured Data:**
```json
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication",
  "name": "AxCouncil",
  "description": "AI-powered decision council platform...",
  "applicationCategory": "BusinessApplication",
  "operatingSystem": "Web",
  "offers": { "@type": "Offer", "price": "0", "priceCurrency": "USD" },
  "aggregateRating": { "@type": "AggregateRating", "ratingValue": "4.8", "ratingCount": "150" }
}
```

**Open Graph Meta Tags (Social Sharing):**
- All tags use absolute URLs (required for Facebook/LinkedIn)
- Image: 1200x630px (optimal for social platforms)
- Description: Concise value proposition

**Client-Side Routing:**
- React Router `createBrowserRouter` with proper routes
- Vercel rewrites all routes to `index.html`
- F5 refresh now maintains navigation state
